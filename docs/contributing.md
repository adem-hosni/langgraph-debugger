# Contributing

Thank you for considering contributing! Please follow these guidelines
to make collaboration smooth.

- Fork the repository and create a topic branch named `docs/*` or
  `feat/*` depending on the change.
- Keep changes focused and provide a clear PR description.
- Run linters and tests locally before opening a PR.
- For documentation changes, update the `docs/` index and any
  relevant code comments.

PR checklist

- [ ] Branch created from `main` (or the repository's default branch).
- [ ] Commits are small and well-described.
- [ ] Tests added or updated for new functionality.
- [ ] Documentation updated.
